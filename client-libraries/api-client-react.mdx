`@workspace/api-client-react` provides auto-generated [TanStack Query v5](https://tanstack.com/query/latest) hooks for every endpoint in the OpenAPI spec. The hooks, query key helpers, and raw fetch functions are regenerated by Orval whenever the spec changes — you do not write them by hand.

<Note>
This package is auto-generated. Do not edit files inside `src/generated/` directly. To add or change an endpoint, update `lib/api-spec/openapi.yaml` and run codegen.
</Note>

## Installation

The package is a private workspace package. Add it to any app in the monorepo:

```json package.json
{
  "dependencies": {
    "@workspace/api-client-react": "*",
    "@tanstack/react-query": "...",
    "react": "^18"
  }
}
```

## Setup

Wrap your application with `QueryClientProvider` before using any hooks.

```tsx src/main.tsx
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import App from "./App";

const queryClient = new QueryClient();

export default function Root() {
  return (
    <QueryClientProvider client={queryClient}>
      <App />
    </QueryClientProvider>
  );
}
```

<Warning>
All hooks throw at runtime if they are rendered outside a `QueryClientProvider`. Ensure the provider wraps your entire component tree before using any hook from this package.
</Warning>

## `useHealthCheck` hook

The primary hook for the `GET /api/healthz` endpoint.

```typescript
export function useHealthCheck<
  TData = Awaited<ReturnType<typeof healthCheck>>,
  TError = ErrorType<unknown>,
>(options?: {
  query?: UseQueryOptions<
    Awaited<ReturnType<typeof healthCheck>>,
    TError,
    TData
  >;
  request?: SecondParameter<typeof customFetch>;
}): UseQueryResult<TData, TError> & { queryKey: QueryKey }
```

| Parameter | Type | Description |
|---|---|---|
| `options.query` | `UseQueryOptions` | Any standard TanStack Query option (`enabled`, `staleTime`, `select`, etc.). |
| `options.request` | `CustomFetchOptions` | Options forwarded to the underlying `customFetch` call (headers, credentials, etc.). |

The hook returns the standard `UseQueryResult` object plus a `queryKey` property for cache invalidation.

### Example: health check component

```tsx src/components/HealthStatus.tsx
import { useHealthCheck } from "@workspace/api-client-react";

export function HealthStatus() {
  const { data, isPending, isError, error } = useHealthCheck();

  if (isPending) return <p>Checking server status…</p>;
  if (isError) return <p>Error: {error.message}</p>;

  return <p>Server status: {data.status}</p>;
}
```

### Example: conditional fetching

```tsx
const { data } = useHealthCheck({
  query: {
    enabled: isAuthenticated,
    staleTime: 30_000,
    retry: 2,
  },
});
```

### Example: custom request headers

```tsx
const { data } = useHealthCheck({
  request: {
    headers: { Authorization: `Bearer ${token}` },
  },
});
```

## Helper functions

### `healthCheck`

The raw async fetch function used internally by `useHealthCheck`. Call it directly when you need to fetch outside of a React component (e.g., in a server action or test).

```typescript
export const healthCheck = async (
  options?: RequestInit,
): Promise<HealthStatus>
```

```typescript
const result = await healthCheck();
console.log(result.status); // "ok"
```

### `getHealthCheckQueryKey`

Returns the stable query key array for the health check query. Use this to manually invalidate or prefetch the cache.

```typescript
export const getHealthCheckQueryKey = () => ["/api/healthz"] as const;
```

```typescript
// Invalidate from outside a component
queryClient.invalidateQueries({ queryKey: getHealthCheckQueryKey() });
```

### `getHealthCheckQueryOptions`

Returns a fully configured options object (`queryKey` + `queryFn`) that you can spread into `useQuery` or pass to `queryClient.prefetchQuery`.

```typescript
export const getHealthCheckQueryOptions = <
  TData = Awaited<ReturnType<typeof healthCheck>>,
  TError = ErrorType<unknown>,
>(options?: {
  query?: UseQueryOptions<Awaited<ReturnType<typeof healthCheck>>, TError, TData>;
  request?: SecondParameter<typeof customFetch>;
}) => UseQueryOptions<...> & { queryKey: QueryKey }
```

```typescript
// Server-side prefetch (e.g. Next.js route handler)
const queryClient = new QueryClient();
await queryClient.prefetchQuery(getHealthCheckQueryOptions());
```

## Updating generated code

<Steps>
  <Step title="Edit the OpenAPI spec">
    Modify `lib/api-spec/openapi.yaml` to add, change, or remove endpoints or schemas.
  </Step>
  <Step title="Run codegen">
    From the workspace root, run the generate script:

    ```bash
    pnpm --filter @workspace/api-spec run generate
    ```

    This overwrites all files in `src/generated/` in both `@workspace/api-client-react` and `@workspace/api-zod`.
  </Step>
  <Step title="Check TypeScript">
    Run a typecheck to confirm nothing broke after regeneration:

    ```bash
    pnpm run typecheck
    ```
  </Step>
</Steps>

<Tip>
See [Codegen](/development/codegen) for a full walkthrough of the generation pipeline.
</Tip>
