The `@workspace/api-server` package is the deployable backend for EduLearn Hub. It runs an Express 5 HTTP server, mounts all routes under `/api`, and uses `@workspace/api-zod` schemas for response validation and `@workspace/db` for database access.

## Package details

| Field | Value |
|---|---|
| Package name | `@workspace/api-server` |
| Entry point | `src/index.ts` |
| App setup | `src/app.ts` |
| Output | `dist/index.cjs` |
| Framework | Express 5 |

## Scripts

```bash
# Start in development mode (tsx, no compilation step)
pnpm --filter @workspace/api-server run dev

# Compile to dist/index.cjs via esbuild
pnpm --filter @workspace/api-server run build

# Type-check without emitting files
pnpm --filter @workspace/api-server run typecheck
```

<Tip>
Run typechecking from the workspace root (`pnpm run typecheck`) to resolve cross-package references correctly. Running `tsc` inside this package alone may fail if `@workspace/db` or `@workspace/api-zod` have not been built yet.
</Tip>

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `PORT` | Yes | TCP port the server binds to. Must be a positive integer. |
| `DATABASE_URL` | Yes | PostgreSQL connection string. Required by `@workspace/db`. |

`PORT` is validated at startup. If the variable is missing or not a positive integer, the process exits with an error before binding.

## Entry point: `src/index.ts`

`src/index.ts` reads and validates `PORT`, then starts the HTTP server:

```typescript src/index.ts
import app from "./app";

const rawPort = process.env["PORT"];

if (!rawPort) {
  throw new Error(
    "PORT environment variable is required but was not provided.",
  );
}

const port = Number(rawPort);

if (Number.isNaN(port) || port <= 0) {
  throw new Error(`Invalid PORT value: "${rawPort}"`);
}

app.listen(port, () => {
  console.log(`Server listening on port ${port}`);
});
```

## App setup: `src/app.ts`

`src/app.ts` creates the Express application, attaches middleware, and mounts the router:

```typescript src/app.ts
import express, { type Express } from "express";
import cors from "cors";
import router from "./routes";

const app: Express = express();

app.use(cors());
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

app.use("/api", router);

export default app;
```

All routes are mounted at the `/api` prefix. The app module is exported separately from the server startup so it can be imported in tests without binding a port.

## Dependencies

| Package | Purpose |
|---|---|
| `express` `^5` | HTTP framework |
| `cors` `^2` | CORS headers |
| `cookie-parser` `^1.4.7` | Cookie parsing |
| `@workspace/db` | Drizzle ORM client and schema |
| `@workspace/api-zod` | Generated Zod schemas for request/response validation |
| `drizzle-orm` | ORM query builder (peer of `@workspace/db`) |
