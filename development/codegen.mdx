## Overview

The API contract lives in a single OpenAPI spec file. [Orval](https://orval.dev) reads that spec and generates two sets of TypeScript artifacts:

<CardGroup cols={2}>
  <Card title="React Query hooks" icon="react">
    Generated into `lib/api-client-react/src/generated/`. Includes a typed fetch client and `useQuery`/`useMutation` hooks for every endpoint.
  </Card>
  <Card title="Zod schemas" icon="shield-check">
    Generated into `lib/api-zod/src/generated/`. Provides runtime validation schemas consumed by the API server.
  </Card>
</CardGroup>

## The OpenAPI spec

`lib/api-spec/openapi.yaml` is the single source of truth for the API contract. All generated code is derived from it — never edit the generated files directly.

```yaml
openapi: 3.1.0
info:
  # Do not change the title, if the title changes, the import paths will be broken
  title: Api
  version: 0.1.0
  description: API specification
servers:
  - url: /api
    description: Base API path
tags:
  - name: health
    description: Health operations
paths:
  /healthz:
    get:
      operationId: healthCheck
      tags: [health]
      summary: Health check
      description: Returns server health status
      responses:
        "200":
          description: Healthy
          content:
            application/json:
              schema:
                $ref: "#/components/schemas/HealthStatus"
components:
  schemas:
    HealthStatus:
      type: object
      properties:
        status:
          type: string
      required:
        - status
```

<Warning>
  Do not change the `title` field in the OpenAPI spec. Orval uses it to derive the output filename (`api.ts`). Changing the title will break all import paths across the codebase.
</Warning>

## Orval configuration

`lib/api-spec/orval.config.ts` defines two output targets:

```typescript
import { defineConfig, InputTransformerFn } from "orval";
import path from "path";

const root = path.resolve(__dirname, "..", "..");
const apiClientReactSrc = path.resolve(root, "lib", "api-client-react", "src");
const apiZodSrc = path.resolve(root, "lib", "api-zod", "src");

// Our exports make assumptions about the title of the API being "Api" (i.e. generated output is `api.ts`).
const titleTransformer: InputTransformerFn = (config) => {
  config.info ??= {};
  config.info.title = "Api";

  return config;
};

export default defineConfig({
  "api-client-react": {
    input: {
      target: "./openapi.yaml",
      override: {
        transformer: titleTransformer,
      },
    },
    output: {
      workspace: apiClientReactSrc,
      target: "generated",
      client: "react-query",
      mode: "split",
      baseUrl: "/api",
      clean: true,
      prettier: true,
      override: {
        fetch: {
          includeHttpResponseReturnType: false,
        },
        mutator: {
          path: path.resolve(apiClientReactSrc, "custom-fetch.ts"),
          name: "customFetch",
        },
      },
    },
  },
  zod: {
    input: {
      target: "./openapi.yaml",
      override: {
        transformer: titleTransformer,
      },
    },
    output: {
      workspace: apiZodSrc,
      client: "zod",
      target: "generated",
      schemas: { path: "generated/types", type: "typescript" },
      mode: "split",
      clean: true,
      prettier: true,
      override: {
        zod: {
          coerce: {
            query: ["boolean", "number", "string"],
            param: ["boolean", "number", "string"],
          },
        },
        useDates: true,
      },
    },
  },
});
```

### Key configuration options

| Option | Value | Description |
|---|---|---|
| `client` | `react-query` / `zod` | Output type: React Query hooks or Zod schemas |
| `mode` | `split` | One file per tag/resource rather than a single monolithic file |
| `clean` | `true` | Deletes the output directory before regenerating |
| `prettier` | `true` | Auto-formats generated files |
| `baseUrl` | `/api` | Prepended to all generated request URLs |
| `mutator` | `custom-fetch.ts` | Custom fetch implementation wrapping the generated client |

## Running codegen

```bash
pnpm --filter @workspace/api-spec run codegen
```

This is equivalent to running `orval --config ./orval.config.ts` from within `lib/api-spec/`.

## Generated output

After running codegen, two files are written:

<Tabs>
  <Tab title="api-client-react">
    **`lib/api-client-react/src/generated/api.ts`** — React Query hooks and fetch functions:

    ```typescript
    /**
     * Generated by orval v8.5.3 🍺
     * Do not edit manually.
     * Api
     * API specification
     * OpenAPI spec version: 0.1.0
     */
    import { useQuery } from "@tanstack/react-query";
    import type {
      QueryFunction,
      QueryKey,
      UseQueryOptions,
      UseQueryResult,
    } from "@tanstack/react-query";

    import type { HealthStatus } from "./api.schemas";

    import { customFetch } from "../custom-fetch";
    import type { ErrorType } from "../custom-fetch";

    /**
     * Returns server health status
     * @summary Health check
     */
    export const getHealthCheckUrl = () => {
      return `/api/healthz`;
    };

    export const healthCheck = async (
      options?: RequestInit,
    ): Promise<HealthStatus> => {
      return customFetch<HealthStatus>(getHealthCheckUrl(), {
        ...options,
        method: "GET",
      });
    };

    export const getHealthCheckQueryKey = () => {
      return [`/api/healthz`] as const;
    };

    // ... query options and useHealthCheck hook
    export function useHealthCheck<
      TData = Awaited<ReturnType<typeof healthCheck>>,
      TError = ErrorType<unknown>,
    >(options?: {
      query?: UseQueryOptions<
        Awaited<ReturnType<typeof healthCheck>>,
        TError,
        TData
      >;
      request?: SecondParameter<typeof customFetch>;
    }): UseQueryResult<TData, TError> & { queryKey: QueryKey } {
      const queryOptions = getHealthCheckQueryOptions(options);
      const query = useQuery(queryOptions) as UseQueryResult<TData, TError> & {
        queryKey: QueryKey;
      };
      return { ...query, queryKey: queryOptions.queryKey };
    }
    ```
  </Tab>

## Adding a new API endpoint

Follow this workflow any time you add a new endpoint to the API:

<Steps>
  <Step title="Define the endpoint in openapi.yaml">
    Add the path, operation, request/response schemas to `lib/api-spec/openapi.yaml`:

    ```yaml
    paths:
      /courses:
        get:
          operationId: listCourses
          tags: [courses]
          summary: List courses
          responses:
            "200":
              content:
                application/json:
                  schema:
                    type: array
                    items:
                      $ref: "#/components/schemas/Course"
    components:
      schemas:
        Course:
          type: object
          properties:
            id:
              type: string
            title:
              type: string
          required: [id, title]
    ```
  </Step>
  <Step title="Run codegen">
    Regenerate all TypeScript artifacts:

    ```bash
    pnpm --filter @workspace/api-spec run codegen
    ```

    Orval writes fresh output to both `lib/api-client-react/src/generated/` and `lib/api-zod/src/generated/`. The `clean: true` option deletes the previous output first.
  </Step>
  <Step title="Implement the route in api-server">
    Create the route handler in `artifacts/api-server/src/routes/`. Import the generated Zod schema for response validation:

    ```typescript
    // artifacts/api-server/src/routes/courses.ts
    import { Router } from "express";
    import { ListCoursesResponse } from "@workspace/api-zod";

    export const coursesRouter = Router();

    coursesRouter.get("/courses", async (_req, res) => {
      const data = await fetchCourses();
      res.json(ListCoursesResponse.parse(data));
    });
    ```
  </Step>
  <Step title="Use the generated Zod schema for validation">
    The generated schema from `@workspace/api-zod` ensures response payloads conform to the OpenAPI contract at runtime. Use `.parse()` to validate and throw on mismatch, or `.safeParse()` for non-throwing validation.
  </Step>
  <Step title="Use the generated React Query hook in frontend components">
    Import the generated hook from `@workspace/api-client-react` in any React component:

    ```typescript
    import { useListCourses } from "@workspace/api-client-react";

    export function CourseList() {
      const { data, isLoading } = useListCourses();

      if (isLoading) return <p>Loading...</p>;
      return <ul>{data?.map(c => <li key={c.id}>{c.title}</li>)}</ul>;
    }
    ```
  </Step>
</Steps>

<Note>
  Never edit files inside `src/generated/` directories by hand. They are overwritten every time codegen runs. Put any customizations in non-generated files (e.g. `custom-fetch.ts` for the fetch client).
</Note>
