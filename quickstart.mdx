This guide walks you through cloning the repository, installing dependencies, connecting a database, and starting the API server.

<Steps>
  <Step title="Check prerequisites">
    You need **Node.js 24** and **pnpm** installed before you begin.

    ```bash
    node --version   # should print v24.x.x
    pnpm --version   # should print 9.x.x or later
    ```

    <Tip>
      Install pnpm with `npm install -g pnpm` or via [pnpm's official installation guide](https://pnpm.io/installation).
    </Tip>
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Fercho12s/EduLearn-Hub.git
    cd EduLearn-Hub
    ```
  </Step>

  <Step title="Install dependencies">
    From the workspace root, run a single install command. pnpm resolves all workspace packages together.

    ```bash
    pnpm install
    ```

    <Note>
      The root `package.json` enforces pnpm via a `preinstall` script. Running `npm install` or `yarn` will exit with an error.
    </Note>
  </Step>

  <Step title="Set environment variables">
    The API server and database layer require two environment variables. Create a `.env` file at the workspace root (or export them in your shell):

    ```bash .env
    DATABASE_URL=postgres://user:password@localhost:5432/edulearn
    PORT=3000
    ```

    | Variable | Description |
    |---|---|
    | `DATABASE_URL` | PostgreSQL connection string used by `@workspace/db` and Drizzle Kit. |
    | `PORT` | Port the Express API server listens on. |

    <Warning>
      Never commit `.env` to version control. Add it to `.gitignore` if it isn't already.
    </Warning>
  </Step>

  <Step title="Push the database schema">
    Apply the Drizzle schema to your database. This introspects the schema definitions in `lib/db/src/schema/` and pushes them directly — no migration files are generated.

    ```bash
    pnpm --filter @workspace/db run push
    ```

    If you hit a prompt about conflicting changes, use the force variant:

    ```bash
    pnpm --filter @workspace/db run push-force
    ```

    <Note>
      `push` calls `drizzle-kit push --config ./drizzle.config.ts` inside `lib/db/`. It requires `DATABASE_URL` to be set in your environment.
    </Note>
  </Step>

  <Step title="Start the API dev server">
    Run the API server in development mode using `tsx` for on-the-fly TypeScript execution:

    ```bash
    pnpm --filter @workspace/api-server run dev
    ```

    You should see output similar to:

    ```text
    Server listening on port 3000
    ```

    The entry point is `artifacts/api-server/src/index.ts`, which reads `PORT` and starts Express. All routes are mounted under the `/api` prefix.
  </Step>

  <Step title="Verify the health endpoint">
    With the server running, confirm it is responding correctly:

    ```bash
    curl http://localhost:3000/api/health
    ```

    Expected response:

    ```json
    { "status": "ok" }
    ```

    <Tip>
      Replace `3000` with whatever you set for `PORT` in your `.env` file.
    </Tip>
  </Step>

  <Step title="(Optional) Run the Pi Day frontend">
    EduLearn Hub includes an interactive Pi Day web application. To start it:

    ```bash
    pnpm --filter @workspace/pi-day run dev
    ```

    Vite will print the local URL (typically `http://localhost:5173`). The app connects to the API server, so make sure the API dev server is already running.

    <Note>
      The Pi Day app is located at `artifacts/pi-day/`. It is built with React 19, Vite, Tailwind CSS v4, Framer Motion, and TanStack Query v5.
    </Note>
  </Step>
</Steps>
